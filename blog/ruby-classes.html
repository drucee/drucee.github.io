Ruby Classes

March 22, 2016

Ruby is what is called an object-oriented programming language. This means that everything in Ruby is some kind of object. We've already seen several types of objects, including strings, integers, floats, arrays, and hashes, but classes are a unique feature of Ruby that can be used to organize objects. Basically, classes are overarching sets of method definitions that can be used for separate instances of that particular class. If this all sounds confusing, think of it this way: for any object of a certain class, there are certain properties that the object has and certain things that we can do to that object. We can think of classes like real-world categories of individual objects, like "dog" or "lamp" or "ticket".

Let's take the class Dog as an example. Notice that all classes begin with a capital letter, by Ruby convention. Classes are instantiated with the following syntax:

class Dog
	    				def initialize(name, breed)
				        @name = name
				        @breed = breed
					    end

					    def name
					        @name
					    end

					    def breed
					        @breed
					    end

						end

As you see here, classes are instantiated with the keyword class and an end keyword. Each class should have an #initialize method defined that can be called to create new instances of the class (new objects of this class).The @ sign in front of the variable name name means that it is an instance variable. We always use instance variables in class methods when we want to be able to access those variables outside of the method definition where they are created. These can be contrasted with local variables, which cannot be accessed outside of the method definition where they were created. The initialize method instantiates instance variables called @name , which we assign to the name and breed arguments that we pass in when we create new Dog objects. Here's how we use this method to create a new Dog object:

new_dog = Dog.new("Rex", "beagle")

You will also notice how we have defined two other methods, #name and #breed, which are accessor methods that will allow us to access attributes of a class object from outside the class by calling these methods on the class object. We can create other methods for our dog object as well, and we define them in the same way we defined the initialize method, within the class Dog.

class Dog
 						def initialize(name, breed)
  						@name = name
  						@breed = breed
 						end
 						def name
  						@name
						end
						def breed
						@breed
						end
						def bark
						puts "Woof!"
						end
						end

Here, we've added the #bark method, which we can call on new_dog or any other Dog object that we may create in the future. Now we can see what happens when we call all of these methods on our new Dog object, new_dog:

new_dog.name
						=> "Rex"
						new_dog.breed
						=> "Beagle"
						new_dog.bark
						=> "Woof!"
