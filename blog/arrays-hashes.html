Arrays and Hashes

March 8, 2016

Ruby gives us two principal ways of creating "container" objects for use in programming. These "containers" are called arrays and hashes. Each of them can be used to store a collection of other types of information, as a restaurant menu might contain different dishes or an address book might contain names and phone numbers, but the ways in which they are used in Ruby differ.

An array is a one-dimensional sort of container object that stores other objects within it, in order. Those objects can be integers, floats, strings, Boolean values, or even other arrays. The order of these objects never changes unless the user explicitly manipulates the order. Arrays are designated by square brackets [].


 menu = ['Hamburger', 'Cheeseburger', 'Double-Double', 'Fries', 'Soda', 'Milkshake']

The above is an example of an array containing strings and stored in the variable menu. There are six strings in the array, each of which is given an index number. Index numbers start from 0 and increase from left to right, such that the first (leftmost) object in an array has index number 0. Using these index numbers we can access any object in the array; if we type in menu[3] to access the fourth object in the array, the console will return that object, Fries.


Hashes are also ordered collections (since Ruby 1.8, anyway) that work very differently from arrays. Instead of storing single objects linearly, hashes store pairs of objects in correspondence, known as key-value pairs. Each key is a unique identifier; you can have multiple keys pointing to the same value, but you cannot have duplicate keys. Hashes are designated by curly brackets {}.


 mn_county_seats = { "Hennepin" => "Minneapolis", "Ramsey" => "Saint Paul", "Dakota" => "Hastings", "Washington" => "Stillwater", "Anoka" => "Anoka", "Scott" => "Shakopee", "Carver" => "Chaska"}

The above is an example of a hash containing strings and using Minnesota county names and county seats as the key-value pairs. We can retrieve values from the hash by typing in the hash's name followed by the key name in square brackets, as follows: mn_county_seats["Dakota"] will return the corresponding value, "Hastings". Hashes are useful to coders when they want to create "dictionaries" or other kinds of libraries of information where one item corresponds to another. Because the name of each key is customizable, hashes are also useful when looking up, selecting or filtering out specific items from a set.
