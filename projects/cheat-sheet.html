Ruby Cheatsheet: Ruby Basics

Ruby is an object-oriented language, meaning its basic unit is the object. Everything in Ruby is considered to be an object; there are no non-object primitives, as there are in non-object-oriented languages like JavaScript.

Ruby Data Types: Numbers

Numbers in Ruby come in two types: integers and floats. Integers are whole numbers, positive or negative; floats have a following decimal point.
Ruby can do mathematical operations (addition, subtraction, multiplication, division, modulo, exponent) on numbers. Doing mathematical operations on integers will produce integer answers, and Ruby will round down in the case of a noninteger answer. (Note that puts writes to the console whatever comes after it).

puts 2 + 4 => 6  
puts 11 - 2 => 9  
puts 3 * 5 => 15  
puts 24 / 7 => 3  # rounded down
puts 9 % 2 => 1  # the remainder of 9 / 2 is 1  
puts 2 ** 4 => 16 # exponent

puts 5.5 + 6.2 => 11.7  
puts 3.5 / 2 => 1.75 # one of the numbers is a float, so float answer

You can use parentheses to perform more complex mathematical operations.

puts 5 * (12.8 + 3 ** 4) / 19.8 - 16 => 7.686868686868685

Ruby Data Types: Strings

Groups of letters, or letters and numbers together, are called strings. Strings are usually designated with single or double quotes, like this: 'This is a string' or "This is a string".
You can add strings together; this is called concatenation. You can also multiply a string by an integer (but you can’t multiply an integer by a string or a string by a string; think about this in terms of natural language, you can say “cow” five times, but you can’t say “five” “cow” times).

puts "monkey" + "shines" => monkeyshines
puts "cow" * 5 => cowcowcowcowcow  
puts 5 * "cow" => <ERROR MESSAGE>

If you want to include an apostrophe or backslash within a string, you have to escape it with a backslash or you will get an error message. For example, to get Ruby to print “What’s up?” you would write puts "What\'s up?".

Variables

Variables are used to store something in a computer’s memory for later use. This process is called assignment. To assign something to a variable in Ruby, you use this syntax (we’re also going to print it to the console here): 

my_name = "Batman" # Ruby convention: use snake_case for variable names  
puts my_name => Batman

Variables can also be reassigned with the same syntax.

my_name = "Batman"  
	puts "No cheese? I'm the goddamn " + my_name + "!" => No cheese? I'm the goddamn Batman!  
	my_name = "Bartman"  
	puts "Everybody do the " + my_name + "!" => Everybody do the Bartman!

Methods

Methods are the actions that you do to numbers, strings, variables, and other kinds of Ruby objects. Using a method in Ruby is referred to as calling that method on an object. To call a method on an object, you append the method’s name to the object with a dot, like this: monkey.climb. That bit of code calls the climb method on the monkey object.
Ruby has several built-in methods for each type of object. The arithmetic expressions we used above are examples of integer/float methods, and here are some examples of string methods:

var1 = "ALEX TREBEK"  
puts var1.length => 11 # characters, including spaces  
puts var1.reverse => KEBERT XELA  
puts var1 => "ALEX TREBEK"

Note that calling the .reverse method on var1 didn’t change what was stored in var1. Many methods do change the variable in place; these are usually known as dangerous methods and have a bang symbol ! at the end of them, by Ruby convention (but neither of these are hard and fast rules!).

var1 = "ALEX TREBEK"  
var1.reverse!  
puts var1 => KEBERT XELA

Writing Your Own Methods

In addition to the built-in methods, you can create your own Ruby methods. Methods can be created by bracketing a block of code within the def and end keywords. Methods can take arguments, which are parameters passed into the method when it is called on an object.

def display_address(name, house_num, street, city, state)
p "Your name is " + name + " " + "and you live at " + house_num.to_s + " " + street + " " + "in the city of " + city + ", " + "in the state of " + state + "."
end  
display_address("Bernie Sanders", 1600, "Pennsylvania Avenue", "Washington", "DC")  
=> Your name is Bernie Sanders and you live at 1600 Pennsylvania Avenue in the city of Washington, in the state of DC.

Some methods take no arguments, like the reverse method above. Ruby will return an error message if the wrong number of arguments are passed into a method.
